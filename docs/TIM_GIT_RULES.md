- Commit small, logical units
- Write clear commit messages
- Do NOT batch unrelated changes

---

## ■ Push Rule

- Push after stable state
- Do NOT leave local-only changes

---

## ■ Pull Rule

- Always pull before execution
- Never assume latest state

---

## ■ Conflict Rule

- Do NOT auto-resolve blindly
- Understand both sides before resolving

---

## ■ Prohibited

- Working outside repo root
- Mixing different project files
- Skipping git checks
